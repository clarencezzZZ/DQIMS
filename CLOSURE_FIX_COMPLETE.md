## 📝 Technical Details

### Why This Happened
PHP 8.0.x (specifically versions < 8.1) has a parser limitation where:
- `?Closure` (nullable type hint) conflicts with `use Closure;` imports
- The parser gets confused and treats "Closure" as an unexpected identifier
- This affects both user code AND vendor packages

### The Solution
Use **fully qualified namespace** instead:
- ❌ `?Closure $param = null` (broken in PHP 8.0)
- ✅ `\Closure $param = null` (works in all PHP versions)

Both approaches are functionally equivalent, but `\Closure` bypasses the parser bug.
