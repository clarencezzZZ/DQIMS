🎯 Section-Wide FIFO Queue System Test

Interactive testing dashboard for First-Come, First-Serve with priority alternation

📋 Test Overview

Test Objective: Verify that the queue management system implements section-wide FIFO with priority alternation.

Rules:
  • Only the earliest waiting queue in the section can be served (FIFO)
  • Serving order alternates: NORMAL → PRIORITY → NORMAL → PRIORITY
  • All service types (categories) within the same section share one unified queue
  • If no priority exists, serve in pure FIFO order
  • Backend validation enforces queue order even if UI is bypassed

Create test inquiries in your database with the following characteristics:

Expected Behavior:
If Inquiry #1 (NORMAL) was just served, then Inquiry #2 (PRIORITY) should be next.
After serving Inquiry #2, Inquiry #3 (NORMAL) should be next.
All Serve buttons except the next one should be disabled!

✅ Step 2: Manual Testing Steps

  1. Navigate to Admin Inquiries Page
    Go to: /admin/inquiries
  2. Observe the Queue Display
    • All inquiries in the same section should be visible together
    • Only ONE inquiry should have an enabled "Serve" button (green)
    • All other waiting inquiries should have disabled buttons (gray)
  3. Test Serving Order
    • Click the enabled "Serve" button for the first inquiry (NORMAL priority)
    • Mark it as "Completed"
    • Observe: The next enabled button should now be for a PRIORITY inquiry (if available)
  4. Test Backend Validation
    • Try to click a disabled "Serve" button on a non-next inquiry
    • You should see a warning message about queue order
    • Even if you use browser dev tools to enable the button, the backend should reject it
  5. Test Priority Alternation
    • Serve a NORMAL inquiry → Next should be PRIORITY (if available)
    • Serve a PRIORITY inquiry → Next should be NORMAL (if available)
    • If only one type exists, serve in FIFO order

🧪 Step 3: Automated Test (CLI)

Run the automated test script to verify the implementation:

php test_section_fifo.php

This script will:

  • Create test categories and inquiries
  • Simulate the serving process
  • Verify the FIFO order with priority alternation
  • Test backend validation

📊 Expected Results

✅ Success Criteria:
  • ✓ Only one "Serve" button is enabled per section at any time
  • ✓ The enabled button belongs to the earliest waiting inquiry (adjusted for priority alternation)
  • ✓ After serving NORMAL, next is PRIORITY (if available)
  • ✓ After serving PRIORITY, next is NORMAL (if available)
  • ✓ Backend rejects attempts to serve out-of-order inquiries
  • ✓ Error messages clearly explain section-wide FIFO rules

🐛 Troubleshooting

Issue: Multiple "Serve" buttons are enabled
Possible Causes:
  • The view is still checking by category instead of section
  • The $nextInquiries array is not properly populated
  • Cache needs to be cleared
Solution: Run php artisan cache:clear and php artisan view:clear
Issue: Backend validation allows out-of-order serving
Possible Causes:
  • The updateInquiryStatus method is not calling the section-wide check
  • The section lookup is failing
    • Solution: Check Laravel logs for errors and verify the section field is populated
    Issue: Priority alternation not working
    Possible Causes:
    • The lastServedType logic is incorrect
    • No completed/serving inquiries to determine last type
    Solution: Ensure there are completed inquiries with different priority types
